Information Architecture for a Website Featuring an Audio-Visual Production & Collaboration App


The Domain

The project domain is “Audio-Visual Content”, focusing on digital audio and moving image. The website aims to bring together a community of Audio-Visual practitioners including artists, academic researchers, students and hobbyists, providing opportunities for learning, skill-sharing, creating, collaborating, showcasing and networking in an open-source environment.

The website consists of an AV web application, which allows the user to create music and video, design sound, use the studio with 5 different set up options* (Audio Production single use, Video Production single use, Audio-Audio Collaboration, Audio-Video Collaboration, Video-Video Collaboration), as well as a Code section to create custom audio and/or video plugins, use them in the web app and share with the community.


Domain Expert Interviews


I conducted remote domain expert interviews with 4 participants.

1-P1, 39: Self-Taught Visual Artist, Music Producer and AV Technician. They create visuals for DJs, clubs internationally, and also perform as a live music act in UK. They have been working as an AV technician at in London.
2-P2, 30: Developer, DJ, Music Producer. They work fulltime as a programmer, and freelance DJ and music producer.
3-P3, 28: Musician, Music Teacher, DJ. Classically trained musician and teacher, multi- instrumentalist and DJ.
4-P4, 30: Animator, DJ, Music Producer. They work fulltime as a senior animator, they produce music as a hobby and use hardware modular synthesizers.

The interviews were followed by a comprehensive domain model which helped outline the components of the project.

Creating the Site Map


The next step was to set the structure of the website by layering all the relevant content in the correct hierarchy. At this stage, I used Optimal Sort platform to conduct a remote Card Sorting exercise. I created 50 labels and asked a group of participants to complete the task. The results were varied to have a concrete idea of a detailed sitemap design, however, it clearly indicated that a separation of Audio and Visuals-related content was preferred by the majority. I created the website structure in line with the card sorting results.

User Flow


To design the user flow, I aimed to create a user journey scenario which would cover at least the most important sections of the website. I decided to focus on a journey of a new user and the likely actions they would take to explore the website.

The scenario: You have been exploring web-based audio-visual tools and your friend has just told you about this new website which has a web-based AV tool and a Code section to design custom plugins. You want to see how it works by registering, logging in, quickly looking at the available resources to understand how the app works, save a project, and try the collaboration option with your friend.

Evaluation


After I completed the wireframes, I was able to evaluate them using the wireframes with 2 participants who had been interviewed.

They were happy with the app section and the general layout, however found the The Library wireframe confusing and the content seemed “too crowded”, as they expressed, and not very friendly to search. This was an important issue since the search function is an important element of The Library, which should allow the user to access the documents. I had to redesign the Library and removed all the sections such as Titles, Categories, Authors, which I had previously included as well as the faceted search options. Instead, I only kept a short Overview and a few ‘featured essays’, and the search functionality. Following the changes on the wireframe, I needed to update the Sitemap as well by removing all the subpages that were previously linked to this page.
As a result, The Library became solely a database of documents which can be searched using the provided options and accessed via the search results.
